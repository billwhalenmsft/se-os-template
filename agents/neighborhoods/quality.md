# ✅ Quality Neighborhood

> The neighborhood that says *"not yet"* — with reasons.

## Residents

| Persona | Role | Specialty |
|---|---|---|
| **Rowan Hayes** | QA Engineer | Edge cases, regression coverage, demo failure modes |
| **Naomi Wells** | Security Reviewer | Auth, data handling, secret hygiene, customer data risk |

**Allied with all three guilds** (AI Craft, Demo Excellence, Customer Empathy) — Quality enforces, the guilds set the bar.

## What this neighborhood owns

- Final QA on every built artifact before it ships
- Security review on anything that touches customer data, secrets, or auth
- Regression coverage on the demo library
- Eval gates on AI components (in partnership with Iris from Build-Time)
- The *"no — and here's why"* response

## Default rhythm

- **Reactive** — triggered by Build-Time handoff
- **Proactive** — periodic audits of the shipped library (does the demo still work? did any environment changes break things?)
- **Pre-flight** — every Friday, sweep what shipped that week to catch what got rushed

## The "no" is a feature, not friction

Naomi's job is to be the *"no"* with reasons. Rowan's job is to find the edge case before the customer does. **Quality slowing something down is the system working correctly.** Skipping Quality is how you ship demos that crash in front of a customer.
