🏭 Ascend SAP Procurement Agent

A native Microsoft 365 Copilot agent that lets procurement managers and approvers interact with SAP ECC 6.0 purchase requisitions using plain English — no SAP GUI, no portal login, no ticket queue.

📱 M365 Copilot Setup

  1. In Copilot Studio → Settings → Channels → enable Microsoft 365 channel, then click Publish
  2. Go to m365.cloud.microsoft/chat → All agents → find Ascend SAP Procurement Agent → Add
  3. Start a new conversation — 6 starter prompt chips appear above the message box
  4. If logo or "Created by Microsoft Solution Engineering" isn't showing: remove and re-add the agent (manifest cache takes ~30 min)

🎤 Key Talking Points

On architecture

"This is Copilot Studio with 8 topics, each connected to a Power Automate flow, which queries Dataverse tables that mirror SAP EBAN, LFA1, and CSKS. In production you replace the Dataverse connector with SAP ECC — everything else stays."

On AI vs. automation

"There's no LLM making procurement decisions. The generative AI handles intent recognition — figuring out which of the 8 topics to call. The actual SAP reads and writes happen through deterministic Power Automate flows. This is safe for enterprise procurement."

On the build time

"8 topics, 6 flows, full Dataverse schema, demo data, M365 Copilot deployment — built in a single sprint using the RAPP pipeline. The same methodology applies to any SAP transaction or D365 workflow."

On adoption

"The agent meets users where they already are — Microsoft 365 Copilot. No training, no new logins, no app install. Procurement managers can start a PR from their morning Teams message."
