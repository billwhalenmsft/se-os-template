The Challenge

SAP Procurement is Powerful.
But Nobody Loves the UI.

🖥️
Context Switching
Procurement managers toggle between SAP GUI, email, Teams, and spreadsheets just to create or approve a single PO. Each switch costs time and increases errors.

Approvers are buried in email. PRs sit pending for days. There's no single view of "what needs my approval right now" without logging into SAP.

Meet your procurement team where they already work — Microsoft 365 Copilot. No new app to learn. Natural language in. SAP data out.

How the Routing Works
ERP_SYSTEM = "SAP_Dataverse"
↓ switch(ERP_SYSTEM)
  "SAP_Dataverse" → Dataverse
  "SAP_ECC"     → SAP Gateway
  "SAP_S4HANA"  → OData REST
↓ All return same schema to CS
Key Design Decisions
  • Copilot Studio topics never change — only the env var switches ERP backends
  • All 7 flows return identical schemas — decoupled from backend
  • On-Premises Data Gateway for RFC/BAPI (SAP ECC)
  • HTTP connector + CSRF tokens for OData (S/4HANA)
  • No custom Azure Functions needed — pure Power Platform
