6   CS Workspace + Channels + Routing 0:05–0:20

CS Workspace Orientation (3 min)

"Before we dive into scenarios, let me orient you to where a Moen agent lives every day. This is Customer Service Workspace — not a portal, not a tab in Teams. It's a purpose-built agent desktop."
Open CS Workspace Home (Tab 1). Walk through the layout:
  • Multi-session tabs — an agent can have 5 cases open at once, each in its own tab. No more lost context when switching.
  • Productivity pane (right side) — agent scripts, knowledge search, Copilot, Smart assist. Always visible.
  • Inbox — unified view of all assigned work items across channels (email, chat, voice). "This is the agent's queue — not Outlook."
  • Agent Home dashboard — KPIs, SLA timers, queue depth at a glance.

Three channels — email, phone, web chat (3 min)

"Janet, you said Moen runs on email + NICE CXone today. Here's how all three channels come together in one system."
Email channel: "When an email arrives at the Moen CS mailbox, D365 auto-creates a case and routes it. The agent sees it in their Inbox and opens it as a session tab. They can reply inline — no Outlook needed." Point at Linda's BBB email in the case timeline — that's a real inbound email they can Reply to.
Phone channel (CXone CIF): "When a call comes in via NICE CXone, the Channel Integration Framework pops a screen with the caller's case history + account context. The agent accepts, and a new session tab opens with the case pre-loaded." We'll demo this LIVE in Hero 1 — Bill calls from +1 (651) 226-3398, screenpop matches Mike Brannigan → Ferguson account → damaged shipment case.
Web chat: "A customer starts a chat on moen.com. It hits a Copilot Studio bot first. If the bot can resolve it (like James Patel's Flo Wi-Fi issue), it does — zero human touch. If not, it transfers to a live agent with the full conversation history." Reference Marie's chat scenario and James Patel's bot-resolved case.

Routing — one workstream, many rule types (4 min)

"This is the part Janet asked about specifically — how do we get the right case to the right agent? Let me show you one workstream that has all the routing options mixed together."
Open the Case workstream: Navigate to Customer Service admin center → Workstreams → click "Case workstream". "This is the single routing workstream that all Moen cases flow through. Inside it, we have classification rules, route-to-queue rules, and assignment rules — stacked. Let me walk you through each."
Classification rules (intent detection): Click the Route to queues tab. Show the rule set. "Rule 1 evaluates the case subject tree — if the subject is 'Complaint / Escalation', it sets priority to High. Rule 2 checks for hot-words in the description (BBB, lawsuit, recall). Rule 3 detects language from the case origin and tags it. These fire in sequence on every new case — the output is a classified, prioritized, language-tagged work item."
Point at the rule conditions editor. Show how each rule is an IF/THEN: IF subject = 'Complaint' THEN set priority = 1 AND add skill 'De-escalation'.
Route-to-queue rules (tier + customer + language mixed): Stay on the same tab, scroll to the queue assignment rules. Walk through how different rule TYPES coexist in one workstream:
Rule typeExampleRoutes to
Customer-basedIF account = "Ferguson" OR "Lowe's" OR "Home Depot"→ Moen-Orders-Strategic-US
Tier-basedIF account category = Preferred (tier 1-2)→ Moen-Claims-US (with 4h SLA)
Language-basedIF language = French→ Moen-Warranty-CA-FR or Moen-Tech-FR
Intent-basedIF subject = "Warranty Claim"→ Moen-Warranty-US-EN
Hot-word escalationIF description contains "BBB" or "lawsuit"→ Moen-Escalation-US (push, not pick)
Default / fallbackNo other rule matches→ Moen-General-US-EN
"These aren't separate workstreams — they're all rules inside ONE workstream, evaluated top to bottom. First match wins. That's why Linda's BBB email went to Escalation, not General."
Skill-based assignment: Click the Assignment rules section. "Once the case is in a queue, assignment rules match it to the best agent based on skills. We created 10 Moen-specific skills — let me show them."
Navigate to User attributes → Skills. Show the Moen skills:
  • Product skills: Faucet and Bath, Shower Systems, Kitchen, Smart Water Flo, Smart Faucet U by Moen
  • Process skills: Warranty Specialist, B2B Claims
  • Language skills: French FR, Spanish ES
  • Escalation skill: De-escalation
"When Marie's French warranty case hits the Moen-Warranty-CA-FR queue, the assignment rule looks for an agent with BOTH 'Warranty Specialist' AND 'French FR' skills. That's a two-skill match — the routing engine finds the best-fit agent automatically."
Push vs Pick: "One more thing — you can set each queue as push or pick. For your escalation queue, we'd recommend push — you don't want a BBB complaint sitting unclaimed. For general inquiries, pick gives agents more control over their workload."
Key demo beat: The room should walk away understanding that one workstream can mix customer rules, tier rules, language rules, intent rules, hot-word rules, AND skill-based assignment — all evaluated in sequence. They don't need 14 separate workstreams for 14 queues.
ROI — Routing: Skills-based routing eliminates the "wrong agent" problem. Industry benchmark: 15-20% of cases are misrouted in skill-less systems. At 70 reps × ~50 cases/day, that's 500-700 misroutes/week eliminated. Each misroute costs ~$12 in handle time + customer frustration → ~$300K-$400K/year in reduced rework.

⭐ Hero 3 — Marie Tremblay: Portal Chat → Bot Self-Service → Agent Handoff 0:55–1:15 · 20 min

Setup — the multi-language "wow"

"Now let me show you the full customer journey — from self-service to live agent — and why you don't need a team of bilingual reps. Marie Tremblay is a homeowner in Québec. She has a leaking Moen faucet and she wants service in French. She goes to your customer portal and starts a chat."
DEMO SEQUENCE: This hero has three acts:
  1. Portal → Bot (French self-service) — Marie chats in French, bot responds in French
  2. Bot → Agent handoff — bot can't fully resolve, escalates with full context
  3. Agent desktop — you see everything translated to English + Intent Agent + Case Mgmt Agent + KB

Act 1: Portal Chat — Bot Self-Service in French

📋 COPY-PASTE SCRIPT: Open the portal chat widget. Copy each "YOU TYPE" message exactly and paste it. Wait for the bot response before sending the next one.
Step 1: Open the portal chat — Open the customer self-service portal. Click the chat widget (bottom-right). The bot greets you. "This is what Marie sees — a branded Moen customer portal with a chat widget."
Step 2: Describe the problem in French
YOU TYPE:
Bonjour, mon robinet Moen fuit à la base. C'est le modèle Genta Posi-Temp dans ma salle de bain. Numéro de série MOEN-MT-2024-77821.
🇬🇧 Translation: "Hello, my Moen faucet is leaking at the base. It's the Genta Posi-Temp model in my bathroom. Serial number MOEN-MT-2024-77821."
Wait for bot response. The bot should confirm the serial and check warranty status.
Step 3: Bot asks about troubleshooting — say you tried it
YOU TYPE:
J'ai déjà essayé de resserrer la base et de vérifier le joint. La fuite continue. Je voudrais parler à quelqu'un.
🇬🇧 Translation: "I already tried tightening the base and checking the seal. The leak continues. I would like to speak with someone."
This triggers the escalation to a live agent. Tell the room: "Marie tried self-service — the bot walked her through troubleshooting, confirmed her warranty. But she needs a human. Watch what happens."

Act 2: Bot → Agent Handoff

Step 4: Watch the handoff — The bot responds:

🇬🇧 Translation: "I'm transferring you to an agent. One moment please."
