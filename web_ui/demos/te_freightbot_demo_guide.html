🚢 FreightBot — TE Connectivity

A Copilot Studio agent that converts forwarder email exceptions into Planner tasks and gives booking agents a shift-start control tower — all through plain English conversation in Microsoft Teams.

🤖 Copilot Studio 📋 Microsoft Planner ⚡ Agent Flows (coming) 📧 Outlook Shared Mailbox 💬 Microsoft Teams

The Problem It Solves

📧 Missed Emails
Forwarder follow-ups arrive asynchronously across shifts and get missed in a shared inbox
🔄 Shift Gaps
No handoff visibility — night shift leaves, morning shift doesn't know what's pending
⚠️ Penalty Risk
Aged exceptions (24–48h) cause shipment delays, demurrage, and customer penalties
🎯 No Ownership
Exceptions sit in a shared inbox — no assigned owner, no SLA tracking

🎤 Key Talking Points

On the business problem "TE Connectivity's freight booking team handles 100+ bookings per week across 5 major forwarders. Each forwarder exception that goes unacknowledged for 24+ hours is a potential shipment delay — and each delay has a penalty exposure. The current process is a shared inbox — that's not scalable."
On the technology choice "We're not adding new tools. This is Microsoft Planner they probably already have, Teams they're already in, and Copilot Studio running on their M365 tenant. Zero net-new tooling."
On Phase 1 scope discipline "We deliberately kept Phase 1 narrow — no automated email replies, no SAP write-back. Get the visibility problem solved first, prove the model, then layer in intelligence. That's how you avoid 18-month projects."
On the shift-start use case "The shift briefing is the killer feature. Every team leader does this manually every morning — scan inbox, count exceptions, figure out what's on fire. FreightBot does it in one message. That's the demo moment."
On Phase 2 ROI "Phase 2 automated acknowledgements alone could save 4–6 hours per day of manual email handling. If a booking agent costs £40k/year, that's meaningful — before you count the penalty cost avoidance."
