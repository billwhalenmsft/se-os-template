Pre-Demo Checklist (Day Before)

Complete 24h before
  • Run provisioning scripts to confirm demo data is in environment
    .\Provision-NavicoDemo.ps1 -Action All then .\Provision-NavicoHeroCases.ps1 -Action All
  • Verify 3 hero cases exist — search for "Simrad NSX 3007", "B&G Triton2", "Lowrance HDS-9 Live" in D365 cases
  • Verify 6 KB articles published — check Knowledge Management in CS admin, confirm KB-SIM-0042, KB-BG-0117, KB-LWR-0088 are published
  • Confirm customer tier field visible on cases — open Atlantic Marine Supply case and verify Platinum Expert tier badge shows
    If not visible, run d365/scripts/13-TierField.ps1
  • Test chat widget — navigate to demo portal URL, confirm chat button is visible and connects to CS workspace
  • Test voice / screen pop — place a test call from Jake's number (+1 651-226-3398), confirm screen pop fires
    If voice is not available, prepare fallback talking point (see fallback section)
  • Confirm Copilot Agent Assist enabled — open a case, check that the Copilot assist panel is visible on the right side of the case form
    Settings: Customer Service admin center → Productivity → Copilot
  • Log in as Emma Thompson — confirm Emma can see the Omnichannel inbox, voice channel, and chat queues
  • Log in as Sarah Mitchell — confirm Sarah can open Omnichannel Insights dashboard
  • Coordinate with John Kassar — confirm demo flow handoff points (Sales context feeds into Service context)

Day-Of Setup (30 min before demo)

30 min prep
  • Open D365 in Chrome — incognito or cleared cache
    Avoid browser autofill interfering with demo fields
  • Open Customer Service workspace as Emma Thompson
  • Pre-open these tabs:
    Tab 1: Customer Service workspace (Emma logged in)
    Tab 2: Atlantic Marine Supply Co. account record (for voice fallback)
    Tab 3: Hero case — "Simrad NSX 3007 — Touchscreen Unresponsive"
    Tab 4: Dealer portal with chat widget (for Scenario 2 customer side)
    Tab 5: Omnichannel Insights dashboard (Sarah's view)
  • Silence all notifications — phone on do not disturb, Teams notifications off
  • Set screen resolution to 1920x1080 — D365 workspace renders best at full HD
  • If presenting via Teams — share the Chrome window (not entire desktop) to avoid accidental tab reveals
  • Have backup screenshots ready — screen pop, Copilot suggest panel, Omnichannel Insights
    Keep in a PowerPoint deck as fallback if live demo fails

Fallback Options

Have these ready
If voice / screen pop isn't live

Open Atlantic Marine Supply account directly. Point to the account record and say: "In a live call, this entire record would have appeared automatically as the phone rang — before I said hello. I'm opening it manually today, but the data and the experience are identical." Then continue the scenario from the account record.

If Copilot Agent Assist panel isn't showing KB articles

Open the KB article manually from the Knowledge Management section. Say: "Let me show you the article that would have appeared here — and in a fully configured environment, Copilot pulls this automatically." Show the article content, then continue.

If chat widget isn't loading on portal

Switch to showing the chat widget from inside Customer Service workspace (test conversation). Alternatively, use a screenshot to illustrate the widget on the portal page and focus the discussion on the routing and Copilot assist experience that follows.

If Omnichannel Insights has no data

Use the Navico-branded screenshot from demo-assets/ as a reference slide. Narrate what each metric represents and tie back to their specific pain points (manual categorization, 1 queue, no visibility).

Recommended Next Steps (propose these in the meeting)

1. Technical discovery — review their current D365 CS config (what's on, what's not)
2. Omnichannel pilot — email + chat for 5 agents, 2 brands, 4 weeks
3. Routing workshop — map Navico's tier/brand/skill matrix to Unified Routing
4. Copilot enablement — same-day config, no code required
5. Voice PoC — RingCentral via CIF adapter, screen pop for 5 agents

⚠️
Key objections to be prepared for

"We already have D365 CS and it doesn't work like that" → It does with proper configuration — same license.
"We're also looking at Salesforce" → You already own D365. Adding SFDC = dual cost + migration risk.
"17 brands is complex" → Start with top 3 by volume. Template is repeatable — 15 min per additional brand.

Manual D365 Config Required (Post-Script)

Not scripted
🔧
These steps require the CS Admin Center — cannot be automated

Complete these before running the demo. Estimated setup time: 2–3 hours.

  • Omnichannel workstreams — create Email, Chat, and Voice workstreams in CS Admin Center → Workstreams
  • Queues — create: Simrad Queue, B&G Queue, Lowrance Queue, C-MAP Queue, Platinum Expert Queue, General Queue
  • Routing rules — configure B2B tier routing (Platinum Expert → Platinum Expert Queue), brand routing (Simrad cases → Simrad Queue), hot word priority boost
  • Chat widget — publish to demo portal page. Ensure pre-chat form collects Account Number for auto-identification
  • Voice channel — configure Teams voice or ACS channel; map Jake's demo number to Atlantic Marine Supply account
  • Copilot features — CS Admin Center → Copilot → Enable: Case Summary, Draft Email, Agent Assist, Real-time translation
  • Customer tier field — run d365/scripts/13-TierField.ps1 to add tier badge to case and account forms
